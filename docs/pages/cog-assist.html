Cognitive Assistance

Supporting Self-Driven Project Based Learning

A part of the selfdriven.ai & heyocto.ai Initatives.

Steps ...

1. Project Design

The learner sets the project objectives, outcomes, and other related elements based on their curiosity.

2. Cognitive Assistance

Using a Cognitive Assistance (CA) tool, the learner creates a project template. The CA tool provides support for organizing the project's goals, objectives, and other elements, helping the learner to create a clear and well-structured plan for their project.

Example using ChatGPT ...

CA Prompt #1: "Create a learning project template for a 14 year old that has: Outcomes, Resources, Tasks and Skills Gained that relates to ocean sustainability"

CA Prompt #2: "Can you now format using this JSON structure: { ... the selfdriven Template Format ... }

3. Cognitive Assistance Output

The output from the Cognitive Assistance tool is copied into a file for checking and verification by the learning-partner.

4. Template Check/Verification

The learning-partner checks the template and uploads it to the selfdriven App.

The template can also be tokenised as an SDI digital-asset (with DID) on the Cardano Blockchain as reference input for any assigned Achievements & Skills. This can be used as a resource for building trust with other learning-partners.

5. Learner Clones & Works On The Project

The learner works on the tasks outlined in the project template, using associated resources to achieve the desired outcomes.

The learning partner supports the learner by acknowledging their progress ("zapping") and providing reflections on growth or suggestions for growth based on their observations.

6. Learning Verification

Once the project tasks are completed, the learning-partner verifies the learner's work based on observations and assigns Achievements and Skills.

Achievements can then be shared within the App/API or on-chain.
